2. End-to-End Dispensing Workflow

Doctor prescribes → Patient pays at Cashier (PHA code issued) → Pharmacy validates code → Stock check → Dispense line-by-line → Patient receives medicines → Chart updated

5
Dispense

Hand to patient

Standard OPD pathway

  1. Doctor adds medication(s) during consultation — dosage, frequency, duration.
  2. Patient pays at Cashier — receives slip with PHA-… service code.
  3. Patient presents PHA code at pharmacy window.
  4. Pharmacist validates code — reviews each medication line.
  5. Check stock on hand; warn if expired or insufficient quantity.
  6. Dispense one line at a time — stock deducted automatically.
  7. Partial Rx: remaining lines stay pending until next visit.

Off-catalog exception

Custom prescriptions with no catalog price require pharmacist to mark available and set unit price before patient pays at Cashier.

Payment gate: In-house medications cannot be dispensed until the order line is paid (unless local policy allows credit for IPD/ER).

5. Validate PHA Code & Dispense

The validate screen is the primary dispensing entry point when a patient presents a payment slip.

  1. Open /pharmacy/validate or click Validate & dispense from hub.
  2. Enter PHA service code (e.g. PHA-7721-M3KP8X2Q).
  3. System verifies payment and loads all medication lines on the code.
  4. Review patient, referring doctor, and prescription status badge.
  5. For each in-house line: check dosage, quantity, stock, expiry.
  6. Click Dispense this medication per line — one at a time.
  7. Add pharmacist notes (batch, substitution, counselling) if needed.

Line status badges

Not yet dispensed Paid · awaiting Dispensed Off-catalog

Prescription progress

Header shows Partially dispensed (2 of 5) or Fully dispensed as lines are completed.

Stock warnings

  • Insufficient stock — option to dispense anyway (stock may go negative for off-catalog)
  • Expired batch — dispense blocked or warning per policy
  • No inventory SKU linked — off-catalog or catalog sync needed

6. Order Alerts

Order alerts (/pharmacy/order-alerts) surface doctor-prescribed medications from consultations, IPD, and Emergency.

  1. Open Order alerts from portal or sidebar.
  2. Review unacknowledged prescriptions needing pharmacy attention.
  3. Each alert shows patient, medication, context (OPD / IPD / Emergency).
  4. Mark as seen when handled via validate flow or off-catalog pricing.

Clinical department alerts strip on portal highlights pharmacy items requiring action.

7. Off-Catalog & Custom Prescriptions

When a doctor prescribes a medication not in the hospital catalog or with zero price:

  1. Line appears on validate screen with Custom Rx or Off-catalog badge.
  2. Pharmacist toggles Available off-catalog if product can be supplied.
  3. Enter unit price (FCFA) and save — patient sent to Cashier on same PHA code.
  4. After payment, re-validate same PHA code to dispense.
  5. Stock may go negative until inventory is updated for off-catalog items.

Cashier billing panel

Cashier OPD bill shows Await pharmacy badge until pharmacist confirms off-catalog availability and price.

Clear off-catalog toggle if product cannot be supplied — patient may need refund or alternative prescription.

8. External Medications

Medications the patient obtains outside the hospital are documented without dispensing:

  1. Doctor marks line as external on prescription.
  2. Validate screen shows external section separate from in-house items.
  3. Pharmacist clicks Acknowledge served outside — no stock movement.
  4. Chart shows line resolved with "served outside hospital" note.

No payment collection at cashier for external-only lines unless local policy requires documentation fee.

12. Nursing Supply Requests

Ward nurses request supplies via Nursing requests (/pharmacy/nursing-requests).

  1. Nurse submits supply request from ward — items and quantities needed.
  2. Pharmacy portal shows badge count when requests pending.
  3. Pharmacist reviews request — mark preparing or dispense.
  4. Stock deducted on dispense; nurse acknowledges receipt on ward.

Integrates with IPD ward board and nursing module supply workflow.

13. Purchase Orders

Pharmacy procurement at /pharmacy/purchase-orders:

  1. New PO — select vendor, add product lines with quantities and unit costs.
  2. Draft — review before confirmation.
  3. Confirm — PO sent to vendor; cannot edit lines freely.
  4. Receive goods — record delivery; stock updated automatically.

Links to procurement vendors and inventory movement audit trail.

Purchase activity report available under Intelligence Reports.

15. Expiry Management

Prevent dispensing expired medications:

  • Expiry report — /pharmacy/reporting/expiry lists batches nearing expiry
  • Validate screen shows expiry hint per order line when batch data linked
  • Dispense blocked or warned for expired stock
  • Remove expired stock via inventory adjustment after physical quarantine

Configure expiry window (days) on expiry report filter.

16. Cashier & Wallet Integration

Pharmacy depends on Cashier for payment verification:

  1. Doctor order creates pharmacy order line — pending payment.
  2. Cashier collects fee; marks line paid; PHA code on receipt.
  3. Pharmacist validates — system confirms paid status before dispense.
  4. Wallet — patient may pay via pre-paid wallet; same PHA validate flow.
  5. Partial payment — patient returns to cashier for remaining lines.

Off-catalog items: pharmacist sets price → cashier bills → patient pays → return to pharmacy.

See Cashier User Manual (/docs/cashier-users-manual) for payment methods and code validity.
