2. End-to-End Imaging Workflow

Doctor orders imaging → Patient pays at Cashier (RAD code issued) → Radiology validates code → Exam scheduled / performed → Results entered (templates) → Finalized → Report on chart & patient copy

Standard OPD pathway

  1. Doctor adds imaging study (X-ray, ultrasound, CT, etc.) during consultation.
  2. Patient pays at Cashier — receives slip with RAD-… service code.
  3. Radiology technician validates code at /radiology/validate.
  4. Exam appears on workflow board — Start exam when patient is ready.
  5. Technician opens radiology template workbench for structured reporting.
  6. Mark complete — result finalized on patient chart and worklist.
  7. Print patient copy from report or worklist action menu.

Order alerts shortcut

Doctor orders also appear on Order alerts (/radiology/order-alerts). Acknowledge → confirm → opens template with patient prefilled.

Payment gate: Radiology cannot process unpaid orders. System blocks validation until Cashier marks the order line as paid.

5. Validate Payment Code

Primary entry when a patient arrives at the imaging desk with a payment slip.

  1. Open /radiology/validate or use validate link from portal/worklist.
  2. Scan barcode or type the RAD service code (e.g. RAD-8821-K2HM9X4P).
  3. System verifies: code belongs to radiology · order line is paid · ticket validity passes.
  4. Validate detail shows patient, referring doctor, and in-house exam lines.
  5. Create radiology request or open template for structured entry.

Validation failures

ErrorCause
Payment requiredPatient has not paid at Cashier
Code belongs to lab/pharmacyWrong department code entered
Expired ticketConsultation-linked validity expired
Duplicate resultReport already registered for this order line

6. Order Alerts

Order alerts (/radiology/order-alerts) surface doctor-ordered imaging from consultations, IPD, and Emergency.

  1. Open Order alerts from portal or sidebar.
  2. Review Needs your attention — unacknowledged orders.
  3. Each alert shows patient, exam name, context (OPD / IPD / Emergency), ordering doctor.
  4. Confirm & open template — prefills patient and order line.
  5. Or Mark as seen if handled via validate-code flow.

Clinical department alerts strip on portal pages highlights radiology items requiring action.

7. Exam Workflow Board

The workflow board (/radiology/workflow) provides a Kanban view of exams in three columns:

Pending
Awaiting start In progress
Exam underway Completed
Report ready

Card actions

  • Start exam — moves to In progress; opens RAD template when service code linked
  • Mark complete — enter findings and conclusion; finalize report
  • Undo — move back to pending queue
  • View report — open finalized report from completed column

Each card shows patient initials, exam name, body part, referring doctor, and appointment date.

For custom exams without linked template code, confirm Start custom report to begin narrative reporting.

8. Radiology Requests

Formal multi-exam requests use Odoo-style request documents.

Create request

  1. Validate RAD service code first (required).
  2. Select patient, prescribing doctor, and scheduled date/time.
  3. Choose imaging room and test group (multi-exam bundle) or individual catalog exams.
  4. Submit — request number assigned (e.g. RAD-REQ-00042).

Request lifecycle

StatusAction
SubmittedRadiology accepts request
AcceptedExams move to in progress on workflow board
In progressPerform imaging; enter results per line
DoneAll exam lines completed — reports available

Open request detail at /radiology/requests/:id for line-by-line progress and room assignment.

9. Templates & Structured Reporting

The Radiology Templates workbench (/radiology/templates) provides structured reporting for X-ray, ultrasound, CT, MRI, and custom exam types.

Opening the workbench

  • From validate screen after code verification
  • From workflow board — Start exam with linked RAD code
  • From order alert confirmation
  • Direct URL: ?code=RAD-…&oi=…&lock=1&autoload=1

Structured entry

  1. Select exam template matching ordered service (modality, body part).
  2. Enter findings — narrative and structured fields where configured.
  3. Select conclusion code (normal, abnormal, follow-up recommended, etc.).
  4. Attach images or documents if PACS integration or upload enabled.
  5. Save draft or finalize — status moves to Results ready.

Test templates maintained under radiology configuration and service catalog.

11. Reports & Printing

View report

/radiology/report/:id — formatted report with facility header, patient demographics, exam details, findings, conclusion, and modality information.

Print patient copy

  • From worklist action menu → Print patient copy
  • From report page → ?print=1
  • Diagnostic handover API for integrated print dialogs

Batch scenarios

When multiple exams share one RAD service code, finalize each line individually; batch print when all lines complete.

Reports use facility branding and premium document theme where configured.

12. Corrections & Recalls

Return for correction

Supervisor with radiology.write can recall a finalized report:

  1. Worklist or registry → Return for correction.
  2. Enter audit reason (required).
  3. Status → Return for correction with revision pending flag.
  4. Re-open via validate code or templates to edit and resubmit.

Recall for edit by code

When linked to billed orders, recall reopens validate-by-code path to maintain payment integrity — same pattern as Laboratory module.

13. External Imaging Results

Studies performed at external facilities are documented separately:

  • Mark order line as external on validate screen
  • Upload report or images via External upload or Attach document
  • Status shows appropriately on worklist
  • Document appears on patient chart as attached diagnostic report

Patient obtains study at referral center; staff uploads PDF or DICOM export when results return.

15. Cashier Integration

Radiology depends on Cashier for payment verification:

  1. Doctor order creates OPD order line with pending payment status.
  2. Cashier collects fee and marks line paid.
  3. System generates RAD-… service code on receipt.
  4. Radiology validate checks paid status and ticket validity.
  5. Partial payments: patient returns to cashier until balance zero.

See the Cashier User Manual (/docs/cashier-users-manual) for payment methods, wallets, and code validity.

16. Rooms, Groups & Catalog

Imaging rooms

/radiology/configuration — define rooms with code, name, modality (X-ray, US, CT, MRI), and active status.

Test groups

Bundle multiple exams (e.g. trauma series, antenatal panel) for single-request workflow.

Service catalog

Imaging studies and prices in Service Catalog → Radiology. Import price lists via /catalog/radiology/import-price-list.

Modalities

Standard modalities: CR/DX (X-ray), US (ultrasound), CT, MR (MRI), RF (fluoroscopy). Room and catalog entries align modality for scheduling and reporting.

Requires radiology.write or service_catalog.radiology.write.
